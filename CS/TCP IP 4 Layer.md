## TCP/IP 4계층
> TCP/IP 모델은 컴퓨터 네트워킹에서 표준 통신규약(프로토콜)을 정의하는 모델 중 하나로, 각 계층마다 네트워크 통신에서 특정한 역할과 책임을 갖고 있다.
- 네트워크 전송시 데이터 표준을 정리한 것이 OSI 7계층, 이 이론을 실제 사용하는 인터넷 표준이 TCP/IP 4계층이다. 즉, TCP/IP 4계층은 실무적이면서 프로토콜 중심으로 단순화된 모델이라고 할 수 있다.
- TCP/IP 4계층의 구조는 네트워크 연결 계층(Network Access Layer), 인터넷 계층(Internet Layer), 전송 계층(Transport Layer), 응용 계층(Application Layer)로 구성되어있다.
- 계층 구분을 통해 각 기능이 서로간의 간섭을 최소화 할 수 있어 유지보수가 편리하다는 특징이 있다.

| TCP/IP 4계층 | 역할 | 데이터 단위 | 전송 주소 | 예시 | 장비 |
| --- | --- | --- | --- | --- | --- |
| 응용 계층 (Application Layer) | 응용프로그램끼리의 데이터 송수신 | Data/Message | - | 파일 전송, 이메일, FTP, HTTP, DNS, SMTP 등 | - |
| 전송 계층 (Transport Layer) | 호스트끼리 송수신 | Segment | Port | TCP, UDP 등 | 게이트웨이 |
| 인터넷 계층 (Internet Layer) | 데이터 전송을 위한 논리적 주소 및 경로 지정 | Packet | IP | IP, ARP, ICMP, RARP 등 | 라우터 |
| 네트워크 연결 계층 (Network Access Layer) | 실제 데이터인 프레임 송수신 | Frame | MAC | Ethernet 등 | 브릿지, 스위치 |

## 네트워크 연결 계층(Network Access Layer / Network Interface Layer)
- **역할** : 이 계층은 실제 물리적 네트워크 인터페이스와 하드웨어를 다루며, 데이터를 전송하기 위한 물리적 연결을 담당한다.
  - 논리주소(IP주소) 가 아닌, MAC주소(하드웨어 주소)를 사용해 기기를 식별한다.
  - 기본적인 에러 검출과 패킷의 프레임화를 담당한다.
- **데이터 단위** : 프레임(Frame)
- **전송 주소** : MAC
- **예시** : 이더넷, LAN 등 

## 인터넷 계층(Internet Layer)
- **역할** : 이 계층은 네트워크 상에서 데이터의 전송을 담당하는 계층으로 서로 다른 네트워크 간의 연결을 담당한다.
  - 데이터 패킷을 **라우팅**하고 목적지로 전달하기 위한 IP주소와 관련된 작업을 담당한다.
    - 네트워크를 연결하고 데이터를 전송하는 기기를 '라우터'라고 하며, 라우터에 의한 네트워크 간의 전송을 '라우팅'이라고 한다. 
    - 라우터가 내부의 라우팅 테이블(Routing Table)을 통해 경로 정보를 등록하여 데이터 전송을 위한 최적의 경로를 찾는데, 이렇게 출발지와 목적지 간의 데이터 전송 과정을 가리켜 End-to-End 통신이라고 부른다.
- **데이터 단위** : 패킷(Packet)
- **전송 주소** : IP
- **예시** : ARP,ICMP,RARP

## 전송 계층(Transport Layer)
- **역할** : 이 계층은 송신자와 수신자 간의 데이터 전달을 관리하며, 데이터의 신뢰성과 흐름을 제어한다.
  - 이 계층의 대표적인 프로토콜로 TCP와 UDP가 있다. TCP는 데이터의 정확한 전달을 보장하는 특징이 있으며, UDP는 빠른 데이터 전송을 제공하지만 데이터 손실 허용한다는 특징이 있다.
- **데이터 단위** : 세그먼트(Segment)
- **전송 주소** : Port
- **예시** : TCP, UDP 

## 응용 계층(Application Layer)
- **역할** : 이 계층은 최종 사용자와 소프트웨어 간의 상호작용을 담당하는 계층이다.
  - 사용자와 가장 가까운 계층이다.
  - 주로 응용 프로그램간의 데이터 교환을 위한 계층이다.
- **데이터 단위** : 데이터(Data), 메시지(Message)
- **예시** : 파일 전송, 메일, 웹 브라우저, HTTP, FTP, SMTP 등
